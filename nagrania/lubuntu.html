Создание чистой ОС для работы с криптой. Linux Lubuntu LiveUSB.


LiveUSB(LiveСD) - это операционная система записанная на CD/DVD диск или на USB флешку. Такая ОС умеет очень быстро запускаться и развертываться без установки на жесткий диск. Загрузившись с LiveUSB вы получите ЧИСТУЮ, безопасную операционную систему для работы с криптовалютными кошельками. Настоятельно рекомендуем при манипуляциях с закрытыми ключами Ваших кошельков использовать такую ОС.

Информация между запусками LiveUSB(LiveСD) не сохраняется, т.е. Вы каждый раз загружаетесь в чистую ОС, без вирусов, троянов, бэкдоров и прочих инструментов злоумышленников.

Для создания LiveUSB мы выбрали ОС Lubuntu, вариант Ubuntu, развивающий более лёгкое, менее ресурсоёмкое, а также энергоэффективное окружение, регулярно обновляется и не требовалтельна к ресурсам ПК.

Мы настоятельно рекомендуем при манипуляциях с закрытыми ключами Ваших кошельков использовать такую ОС.

Каждый биткойн-адрес имеет два ключа: «открытый ключ» (“public key”) и «закрытый ключ» (“private key”).

Биткойн-адрес вычисляется из открытого ключа, и этот адреса вы можете безопасно распространять для получения на него платежей.

Подумайте об этом, как о том чтобы поделиться своим адресом электронной почты с кем-то: они смогут отправить вам электронное письмо, но не смогут попасть в ваш почтовый ящик, чтобы отправлять почту от Вашего имени. Точно так же никто не может попасть в кошелек и взять с него биткойны с открытым ключом. Из биткойн адреса нельзя получить не открытый не тем более закрытый ключ.

Закрытый (приватный) (“private key”) ключ - это то, что позволяет пользователям управлять биткойнами, и это то, что нужно ЗАЩИЩАТЬ, чтобы сохранить Ваши биткойны в безопасности. Тот, кто держит закрытый ключ, считается владельцем биткойнов по этому адресу.

Существует несколько различных методов, которые пользователи используют для защиты своих секретных ключей. Чтобы сохранить закрытый ключ, можно зашифровать свой кошелек с биткойнами с помощью частного пароля, но это, как правило, самый базовый уровень безопасности , который может быть нарушен компьютерными хакерами или вирусами. Другие предпочитают хранить свои закрытые ключи в автономном режиме без подключения к сети - холодное хранение.

В качестве другого подхода к защите многие пользователи используют адреса с мультиподписью, которые позволяют нескольким сторонам удерживать часть адреса в ключе или удерживать один из многих ключей, подключенных к одному адресу. Когда один пользователь хочет получить доступ к биткойнам, эти другие держатели также должны будут одобрить транзакцию. Количество подписей может быть настроено, и пользователи могут настроить его таким образом, чтобы несколько проверок предоставлялись отдельными устройствами, каждый из которых управлялся отдельно.

Среди множества опций, доступных для защиты секретных ключей кошелька биткойн, у каждого есть определенные плюсы и минусы, которые пользователи должны будут взвесить. Важно следить за тем, чтобы ваши инвестиции были защищены таким образом, чтобы предоставить вам доступ, когда вам это нужно.
